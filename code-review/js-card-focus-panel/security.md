# Security Review — Card Focus Panel
Lines: 1925–1994 | File: public/mtg-commander.html

## Findings

### `focus-actions` innerHTML built with unsanitized `idStr`
**Severity:** Low
**Lines:** 1945–1952
`idStr = JSON.stringify(id)` is inserted into inline `onclick` attributes. Since IDs are generated by `Date.now()` or sequential integers (game engine internals), this is safe. However, the pattern should not be extended to user-supplied data.
**Action:** No immediate change required, but document that `idStr` must always be a serialized integer or UUID, never user text.

### `selectHandCard` inserts raw numeric `idx` into onclick attributes
**Severity:** Low
**Lines:** 1987–1989
`${idx}` is a JavaScript array index (always an integer), so there is no injection risk here. The pattern is safe.
**Action:** No change needed for security, though using a stable ID (see static review) is preferred.

### `focus-oracle` set via `textContent` — safe
**Severity:** Low
**Lines:** 1942, 1981
Oracle text from Scryfall is set using `textContent`, not `innerHTML`, which prevents any HTML injection from card text (e.g., reminder text containing `<i>` tags won't render, but there's no XSS risk).
**Action:** No change needed. If formatted oracle text is ever desired, switch to a sanitized innerHTML approach.

## Summary
No significant security issues. The use of `textContent` for oracle text is correct. The idStr and idx interpolation into onclick attributes are safe because both values are numeric. The pattern should be guarded against future changes that introduce string IDs from external sources.
