# Security Review — Render Play Area
Lines: 1858–1923 | File: public/mtg-commander.html

## Findings

### `bfCardHTML` uses `escapeHtml` for card names — correct, but `bfc.id` is interpolated raw into `onclick`
**Severity:** Medium
**Lines:** 1884, 1889, 1895
`bfc.id` is serialized via `JSON.stringify(bfc.id)` and embedded directly into an `onclick` attribute: `onclick="selectBFCard(${idStr})"`. Since IDs are generated by `crypto.randomUUID()` elsewhere in the file (or are database UUIDs), they should only contain hex digits and hyphens. However, if an ID were ever sourced from a non-UUID value (e.g. a crafted API response), the `JSON.stringify` would not prevent script injection in an `onclick` context — JSON.stringify of a string like `"')};alert(1)//"` would still produce a malicious attribute.
**Action:** Validate that all `bfc.id` values are UUIDs before use, or use a `data-id` attribute with a `dataset` lookup in the event handler instead of inline `onclick`.

### Card image `src` attribute uses Scryfall CDN URL without validation
**Severity:** Low
**Lines:** 1883, 1913
Image URLs from `card.image_uris?.small` and `card.image_uris?.normal` (from Scryfall API) are placed directly into `src` attributes. While Scryfall URLs are trusted in this context, no validation ensures the URL begins with the expected Scryfall CDN domain. A compromised API response could inject a different URL.
**Action:** Low risk in practice. Optionally add a URL prefix check: only use the image if it starts with `https://cards.scryfall.io/`.

### `bfc.card.type_line` interpolated via `?.split('—')[1]?.trim()` without escaping
**Severity:** Medium
**Lines:** 1899
The token fallback type display uses `bfc.card.type_line?.split('—')[1]?.trim()` interpolated directly into innerHTML without `escapeHtml`. If `type_line` contains a `<` or `>` character (unusual but possible in Scryfall data), this would inject unescaped HTML.
**Action:** Wrap with `escapeHtml(...)`: `escapeHtml(bfc.card.type_line?.split('—')[1]?.trim() || 'Token')`.

### Inline cost display in `renderPlayHand` not escaped
**Severity:** Low
**Lines:** 1920
`costShort` (the output of `formatManaCostShort(card.mana_cost)`) is interpolated directly into innerHTML. `formatManaCostShort` strips `{` and `}` but does not escape HTML special characters. A mana cost string like `{<script>}` would survive the regex replacement as `<script>`. While Scryfall mana costs are not expected to contain `<`, the lack of escaping is a pattern violation.
**Action:** Apply `escapeHtml(costShort)` before interpolating into the inline style div.

## Summary
Two medium-severity issues: `type_line` in token cards is interpolated without `escapeHtml`, and `bfc.id` in `onclick` handlers relies on UUID format rather than structural safety. Two lower-severity issues involve unescaped `costShort` and unvalidated image URLs. The correct `escapeHtml` usage on card names is commendable.
