# Security Review — Render Play Area
Lines: 1858–1923 | File: public/mtg-commander.html

## Findings

### Card image URLs injected directly into `src` attribute without validation
**Severity:** Low
**Lines:** 1890, 1913, 1918
`bfc.card.image_uris?.small`, `card.image_uris?.normal`, etc. are injected directly into `<img src="...">` without validation. These URLs come from Scryfall API responses stored in memory. If the Scryfall data were compromised or a malformed card object were injected, an attacker could load arbitrary image URLs (used for tracking/SSRF in browsers) or attempt `javascript:` URIs.
**Action:** Validate that image URLs start with `https://` before injecting, or use a URL allowlist for `cards.scryfall.io`. This is a low-severity concern given the Scryfall data is fetched over HTTPS and not user-supplied.

### `bfCardHTML` injects `bfc.id` via `JSON.stringify` into `onclick`
**Severity:** Low
**Lines:** 1884, 1889, 1895
`JSON.stringify(bfc.id)` is used to safely embed the ID in the onclick attribute. Since `bfc.id` is a UUID generated by the application (not user input), this is safe. If the ID scheme ever changed to user-supplied strings, `JSON.stringify` alone would not prevent XSS in attribute context.
**Action:** No immediate action. Document that `bfc.id` must remain a developer-controlled value (UUID).

### `escapeHtml` is used correctly for name rendering
**Severity:** Low
**Lines:** 1891–1902
`escapeHtml(bfc.card.name)`, `escapeHtml(shortName)` — card names from Scryfall are properly escaped before innerHTML injection. This is correct usage of the codebase's XSS prevention pattern.
**Action:** No action needed.

### Inline style in `renderPlayHand` injects mana cost string via `formatManaCostShort`
**Severity:** Medium
**Lines:** 1920
`${costShort}` is injected into a `<div>` innerHTML without `escapeHtml`. `costShort` comes from `card.mana_cost` processed by `formatManaCostShort`, which strips `{}` brackets and returns a substring. The result contains only alphanumeric mana symbols (W, U, B, R, G, C, X, digits), so XSS is not feasible in practice. However the value is not escaped before injection.
**Action:** Wrap `costShort` in `escapeHtml()` before injecting into innerHTML to be consistent with the codebase's XSS prevention policy.

## Summary
The main actionable finding is that `costShort` is injected into innerHTML without `escapeHtml`. While the current data path makes exploitation infeasible, it violates the project's policy of always escaping user/API data before innerHTML injection.
