# Security Review — Context Menu
Lines: 1996–2021 | File: public/mtg-commander.html

## Findings

### `id` passed to `showCtxMenu` stored as `ctxTarget` — no validation
**Severity:** Low
**Lines:** 1999
`ctxTarget = id` stores the `id` argument directly. Since `id` is passed from an onclick attribute on battlefield cards (generated by the render functions), it originates from `bfc.id` which is always `Date.now() + Math.random()` (a number). No validation occurs, but there is no injection risk in this specific case.
**Action:** No immediate change needed; document that `ctxTarget` is always a numeric ID.

### Menu position set via `style.left`/`style.top` with user-influenced values
**Severity:** Low
**Lines:** 2004–2005
The menu position is derived from `e.clientX` and `e.clientY` (mouse coordinates), passed through `Math.min()` and then set as `style.left`/`style.top`. Since these are numbers from a browser event (not from user-supplied text input), there is no injection risk here.
**Action:** No change needed. This is safe.

## Summary
No security issues found. All values flowing through this section are numbers (mouse coordinates and card IDs), not user-supplied text, so there is no XSS or injection risk.
